## Coding Philosophy

**Simplicity Over Cleverness**: Prefer readable solutions over clever ones. If a junior engineer can't follow it, simplify.

**Leverage Existing Patterns**: Search the codebase first. Consistency with what's already there trumps a marginally better approach.

**Evidence-Based Decisions**: Measure before optimizing. Verify with data, not intuition.

**Minimal Abstraction**: Only abstract when patterns repeat 3+ times. Three similar lines beat one premature abstraction.

## Workflow

1. **Discover** — Read existing code for patterns, naming, architecture before writing anything
2. **Match** — Follow the style and structure of surrounding code
3. **Verify** — Run tests and linters before considering anything complete

## Communication
